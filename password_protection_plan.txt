# Implementation Plan: Password Protected Surprises 🔓🎁

This plan outlines the steps to add a "Surprise Password" feature across all Pookie Wishes templates. This adds a layer of exclusivity and fun, requiring the recipient to enter a custom password set by the sender before revealing the surprise.

## User Review Required

> [!IMPORTANT]
> **Client-Side Security:** This password protection is implemented as a front-end "gate." While it prevents casual viewing, it is intended for fun and interactivity, not high-security data protection.
> 
> **Ease of Use:** Senders will be prompted to set an optional password during the personalization step. If left blank, the surprise remains public-accessible as it is now.

## Proposed Changes

### [Component] Data Schema (site.json)

#### [MODIFY] [site.json](file:///d:/Cipher/pookie-v3/data/site.json)
- Add a new "Set Surprise Password" field to the `submitDetails` array for all templates.
- This will automatically enable the password input in the `order-success.html` personalization form.

---

### [Component] Global Template Loader

#### [MODIFY] [template-loader.js](file:///d:/Cipher/pookie-v3/js/template-loader.js)
- Update `window.getPookieData` to detect if a `password` property exists in the order data.
- Implement a `showPasswordGate(correctPwd)` function that:
    - Injects a beautiful overlay with a 3D-effect Gift Box.
    - Provides a password input field.
    - Uses `canvas-confetti` for a celebration when unlocked.
    - Blocks the rest of the template script until the password is correct.

---

### [Component] Visual Design / Aesthetics

#### [NEW] [password-gate.css](file:///d:/Cipher/pookie-v3/css/password-gate.css)
- Create a premium design for the locked state:
    - Floating/bobbing gift box animation.
    - Glassmorphism input field.
    - Smooth transitions when the box "explodes" open.
