 ─────────────────────────────────────────────
 3.7 CHECKOUT FLOW
 ─────────────────────────────────────────────
 File:  pages/checkout.html

 FREE ORDER FLOW (total = Rs.0):
   1. Write buyer to buyers/{email} (stage 1 or 2)
   2. Increment counters/stats.uniqueBuyersCount
   3. Get sequential order ID via ATOMIC TRANSACTION ✅ (fixed 14 Apr)
   4. Save to orders/{orderId} with status: 'paid'
   5. Redirect to order-success.html

 PAID ORDER FLOW:
   1. Call /api/razorpay-order backend -> get razorpay order_id
   2. Write buyer to buyers/{email}
   3. Get sequential order ID via ATOMIC TRANSACTION ✅ (fixed 14 Apr)
   4. Save order as 'pending_payment'
   5. Open Razorpay popup modal
   6. On success -> update to 'paid' -> redirect to order-success.html

 Status: WORKS ✅
 BUG: No server-side payment signature verification (Razorpay webhook missing).
 BUG: Cart price is client-side only — user can edit localStorage before checkout.
 BUG: syncCheckoutCart() checks email field on load, but field is empty at load time,
      so quota sync always falls back to counter estimate, not user-specific quota.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 BUG #1 [CRITICAL] ORDER OVERWRITE - FIXED ✅
   Cause: admin updateStats() overwrote counters/stats.totalOrders with the
          count of paid orders visible in current session. If only 2 orders
          loaded, counter became 2. Next new order got ID "order3" which
          already existed -> overwrote a real customer's order.
   Fix:   Removed totalOrders write from admin panel entirely.
          Counter is now ONLY written by checkout via atomic transaction.

 BUG #2 [CRITICAL] ORDER ID RACE CONDITION - FIXED ✅
   Cause: Sequential ID generated in 3 steps: read -> compute -> write.
          Two simultaneous orders could get same ID and overwrite each other.
   Fix:   Replaced with Firestore runTransaction() - atomic and safe.

 BUG #3 [HIGH] PERMISSIONS ERROR ON ORDER SAVE - FIXED ✅
   Cause: Firestore rules had no rule for buyers/ collection.
          Unauthenticated checkout writes were denied by default.
   Fix:   Added buyers collection rule. User must apply updated rules
          in Firebase Console -> Firestore -> Rules.

 BUG #4 [HIGH] SUGGESTIONS NOT SHOWING IN ADMIN - FIXED ✅
   Cause: nikhilrawat2005@gmail.com missing from Firestore rules for
          suggestions/ and users/ collections -> read denied -> empty list.
   Fix:   Updated Firestore rules (apply in Firebase Console).
          Added orderBy fallback in loadSuggestions() for resilience.
