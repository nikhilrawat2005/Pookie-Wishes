REMAINING BUGS & ISSUES LIST
Project: Pookie Wishes
Date: 2026-05-09

1. STICKER SYSTEM (PBOX) INCONSISTENCY
--------------------------------------
The "Use Default Stickers" option is only available for 4 templates (Hello Kitty, Harry Potter, Love Trap, Sorry). 
Other photo-based templates do not have this fallback. If a user doesn't have photos for those, they might get stuck.

2. PATHING ISSUE FOR STICKERS
-----------------------------
In order-success.html, stickers are mapped to "../../assets/". While this works for delivered templates, 
some templates might have local pathing issues or conflict with the server root depending on where they are hosted.

3. REDUNDANT INITIALIZATION (Anniversary Journey)
-------------------------------------------------
In templates/anniversary-journey-template/script.js, initData() and buildPhotoMoments() are called twice 
(once at the top and once at the bottom of the script). This causes redundant Firestore hits.

4. WEDDING TEMPLATE DATA MISMATCH
---------------------------------
site.json says 'hasPhotos: false' for Wedding Wishes, but the template code and template-loader.js 
contain references to photo arrays. This needs to be synchronized.

5. PRICING ENGINE REDUNDANCY
----------------------------
The May Special Offer logic (0 price) is duplicated in js/app.js and pages/checkout.html. 
If the promotion date changes, it needs to be updated in both files manually.

6. HARDCODED PLACEHOLDERS
-------------------------
js/template-loader.js contains some hardcoded placeholders (like "Dagabaaz Re" song or "Sahil & Priya") 
which appear if Firebase fails to load, but they might not match the template's aesthetic perfectly.
